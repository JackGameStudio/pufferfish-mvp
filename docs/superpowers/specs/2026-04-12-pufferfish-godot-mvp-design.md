# 弹弹河豚 × Godot 4 重设计文档

> 日期：2026-04-12
> 状态：已确认，待实施

---

## 1. 项目目标

将弹弹河豚 MVP 从 HTML5 Canvas 迁移至 Godot 4，保留核心弹跳玩法，同时升级架构、美术（像素风）和音效。

**MVP 定义：** 从起点弹到终点，途中碰撞圆形障碍扣血，HP 归零则失败。

**范围终点：** 完整 MVP（架构 + 像素美术 + 音效 + 粒子），但**先交付一个可玩关卡**再扩张。

---

## 2. 核心技术选型

| 模块 | 选择 | 理由 |
|------|------|------|
| 引擎 | Godot 4.6+ | 跨平台（PC/手机/Web），代码 90% 共用 |
| 渲染 | 2D + Pixel Art | 8-bit/16-bit 复古风格 |
| 关卡编辑 | 代码生成节点（对应原版纯数据关卡） | 与原版数据结构一致，无需 TileMap 编辑器 |
| 物理 | CharacterBody2D + 自定义摩擦力 | 完全代码控制，匹配原版手感（无重力，靠摩擦减速） |
| 导出平台 | Windows + macOS + Android + iOS + HTML5 | Godot 一键导出 |

## 3. 像素规格（阶段零，必须先定）

| 参数 | 值 | 说明 |
|------|-----|------|
| PPU（像素/单位） | **32** | 行业常用值，2的幂次方便缩放 |
| 鱼精灵图尺寸 | **64×64 px** | 按最大膨胀状态（36px）设计，留余量 |
| 鱼碰撞半径 | **20px = 0.625 单位** | 对应原版 `fish.radius = 20` |
| 最大膨胀半径 | **36px = 1.125 单位** | `20 × 1.8 = 36` |
| 瓦片尺寸 | 32×32 px | TileMap 标准格（如果用 TileMap） |
| 关卡逻辑尺寸 | **4096×2048 px = 128×64 单位** | 新尺寸（边界扩大） |
| 障碍物 | 保持原版像素尺寸 | 不等比放大，障碍密度不变 |

> ⚠️ **阶段零必须完成后再写代码。** 鱼精灵图和关卡尺寸是所有物理计算的基础。

## 4. 核心机制（严格对标原版 index.html）

### 4.1 鱼的物理参数

| 参数 | 原版值 | 说明 |
|------|--------|------|
| `maxScale` | 1.8 | 膨胀上限倍率 |
| `inflateRate` | 0.8/s | 按住1秒满膨胀 |
| `deflateRate` | 1.4/s | 松手0.5秒缩回 |
| `launchVel` | 900 | 弹射初速（px/s） |
| `friction` | 0.992 | 每帧摩擦衰减 |
| `rotSpeed` | 6 rad/s | charging 时自转速度 |
| `invincTime` | 0.5s | 受伤无敌帧 |

### 4.2 状态机

```
idle ──(按住)──→ charging ──(松开)──→ flying
  ↑                                     │
  └────────(速度<5)─────────────────────┘

charging: scale增大(1→1.8) + rotation自转(6rad/s)
flying:   弹弓弹射(rotation+PI方向) + 摩擦减速 + rotation对齐速度
          flying中按住也会触发charging（charging优先）
idle:     速度≈0，等待输入
```

### 4.3 碰撞规则

- **圆形障碍**：物理推开 + 扣血（`damage = 10 + (scale-1)*40`）+ 无敌帧
- **矩形障碍**：物理推开，**不掉血**
- **膨胀无敌**：scale ≥ 95% 时完全无敌

### 4.4 胜利/失败

- 胜利：到达终点区域（半径40px）
- 失败：HP ≤ 0（初始100）

## 5. 项目结构

```
res://
├── project.godot
├── scenes/
│   ├── main.tscn              # 游戏入口
│   ├── levels/
│   │   └── level_01.tscn     # 第一关（代码生成障碍物）
│   ├── player/
│   │   └── fish.tscn          # 河豚角色
│   ├── obstacles/
│   │   ├── circle_obstacle.tscn  # 圆形障碍（掉血）
│   │   └── rect_obstacle.tscn    # 矩形障碍（不掉血）
│   └── ui/
│       └── hud.tscn           # HP条（屏幕空间）
├── resources/
│   ├── sprites/               # 像素精灵图
│   │   ├── fish.png             # 64×64 px
│   │   └── obstacles/           # 障碍精灵图
│   └── audio/                 # 音效
│       ├── bounce.wav
│       ├── hurt.wav
│       └── win.wav
└── scripts/
    ├── fish.gd               # 核心：状态机+充气+弹射
    ├── game_manager.gd       # win/gameover 状态
    ├── camera_follow.gd      # 摄像机跟随+震动
    ├── circle_obstacle.gd    # 圆形障碍碰撞
    ├── rect_obstacle.gd      # 矩形障碍碰撞
    ├── level_loader.gd       # 关卡数据→节点生成
    └── hud.gd                 # HP条更新
```

## 7. 验收标准（对标原版逐项）

| # | 检查项 | 原版行为 |
|---|--------|----------|
| 1 | 按住充气 | scale 1→1.8，旋转，松开后缩回 |
| 2 | 弹弓弹射 | 松手沿 rotation+PI 方向飞出，900*power |
| 3 | 摩擦减速 | velocity *= 0.992，速度<5停止 |
| 4 | 圆形障碍 | 推开+扣血+无敌帧（scale≥95%时无敌）|
| 5 | 矩形障碍 | 推开不掉血 |
| 6 | 摄像机跟随 | 平滑 lerp 0.08 |
| 7 | 过关 | 到达 endZone 显示胜利 |
| 8 | 失败 | HP=0 显示游戏结束 |
| 9 | 重置 | 点击/空格重新开始 |
| 10 | Windows 导出 | .exe 独立运行 |
